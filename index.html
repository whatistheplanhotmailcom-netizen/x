Troubleshooting
If you hear nothing on iOS:
• Check the silent-mode switch on the side of the phone — flip it OFF (no orange).
• Tones use Web Audio which is muted by the silent switch. Voice is NOT muted.
• Turn up the volume buttons (Ringer volume, not Media).

If voice doesn't work:
• iOS may need a few seconds to load voices on first launch.
• Wait, then re-open Sound check.
• Some voices need download in Settings → Accessibility → Spoken Content.

If tested in an in-app browser (TikTok/Facebook):
• Open the page in real Safari or Chrome instead.

Alerts
Alert engine
Legacy · current alert engine controls real alerts (default). Shadow · legacy still controls alerts; intelligence engine evaluates in parallel and logs decisions. Active · intelligence can suppress legacy alerts on low-confidence cases. Trusted observations (confidence ≥ 3 or 3+ confirmations) are protected from suppression except on strong direction mismatch.
One-tap restore of the original alert engine. Use this immediately if Shadow or Active mode does anything unexpected while you're driving — it's always available, and Legacy is the safe fallback.
Sound

Voice type
Chooses which system voice speaks. None disables all speech (tones still play if Sound = Tone).
Announce side in voice

Alert distances (meters)
Tap a chip to remove · alerts fire when you cross each distance
An alert fires each time you cross one of these distance rings approaching a point. More rings = more warnings; fewer = less noise. Only fires for the closest point ahead, so two nearby cameras don't double up.
Flash border at (m)
Distance at which the Next ahead card on the main screen pulses red. Purely visual — independent of audio alerts.
Repeat alert × times
How many times each distance announcement repeats. 1 = say it once. 3 = say "Speed cam in 500m" three times in a row.
Repeat gap (sec)
Pause between repeats of the same alert. Smaller = more urgent rapid fire; larger = breathing room.
Tone frequency (Hz)
Pitch of the proximity-ping tone. Higher = sharper / more cutting; lower = softer. Tap to preview after changing.
Proximity ping start (m)
Distance at which the continuous beep starts. Cadence speeds up automatically as you close in: slow (~1.2s) ≥ start, faster (~0.5s) ≥ start/2, fastest (~0.2s) within start×0.2.
Proximity ping
Master toggle for the continuous countdown ping. Threshold-cross alerts (above) still fire when this is off.
Sound Alerts
Per-sound Frequency controls how the Try preview plays — High = 3× firm, Medium = 2× normal, Low = 1× soft. Used For uses real capture type keys (Speed cam, Speed zone, …) and reserved system targets. This panel is preview / mapping only; live alert triggering is unchanged.
Speed limit alerts
Mode
What plays when you're driving faster than the current speed limit + the threshold below. Off = silent (red LIMIT sign still flashes). Beep/Voice/Both = audio cue when you cross the limit.
Trigger when over by (km/h)
How many km/h over the limit before the alert fires. 5 means a 100 km/h zone won't alert until you hit 105.
Revalidate stored speed limits
When ON, as you approach a saved speed-limit sign the app may ask "The speed limit here is N. Confirm?" Tap YES to strengthen the observation, NO to record a rejection. Default OFF — never prompts unless explicitly enabled. One prompt per point per trip; 30-minute cooldown across trips.
Here-now announcement
Says "X is here" the moment you arrive at a captured point. Once it fires, the regular distance announcements ("in 500 m") for that point are suppressed to avoid double-ups.
Speed threshold (km/h)
Above this speed → triggers within 100 m of the point (more reaction time at highway speed). Below → within 50 m (tighter ring for city driving).
Say × times
How many times "X is here" repeats in one announcement (concatenated as "X is here. X is here.").
Trips
Records distance driven, duration, and max speed for each session. Independent of Start GPS — start a trip to log, start GPS to track. Most users want both on.
GitHub backup
⚠ Token is stored in browser localStorage in plain text. Use a fine-grained personal access token with write access to one private repo only. Anyone with physical access to this device's storage can read it.
Token
GitHub fine-grained PAT. github.com → Settings → Developer settings → Personal access tokens → Fine-grained. Permissions: Contents = Read & write on the target repo.
Repo
Format: owner/repo. e.g. jane/road-alert-backup. Repo must already exist.
File path
Filename inside the repo. Created on first backup. Can include subfolders: backups/jane.json.
Auto-backup every 5 min
When ON, the app pushes a fresh snapshot to GitHub every 5 minutes if data has changed since the last backup. Status shown in the green ticker on the bottom of the main screen.
No backup yet
Backup now pushes immediately, ignoring the change-check. Restore pulls the remote file and OVERWRITES local data (asks for confirmation first).
Reset database clears all local points + destinations on this device (keeps settings + GitHub config). Your GitHub backup is left untouched, so you can Restore to undo. New database clears local data AND pushes the empty database to GitHub, replacing the backup with a clean slate. Both ask for confirmation.
Data

Export/Import: download/upload all data as a JSON file (manual backup outside of GitHub). Edit JSON: directly edit the underlying data structure — power-user tool, can break things.
Cache
Force the browser to drop cached JS/CSS and fetch the latest version. Use this if a new release doesn't appear after a normal refresh. Your saved data (destinations, points, settings, tokens) is not affected.
